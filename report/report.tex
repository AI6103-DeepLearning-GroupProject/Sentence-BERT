\begin{abstract}
This project revisits the original Sentence-BERT (SBERT) recipe on the legacy
sentence-transformers v0.2.4 codebase (Dec 2019) and evaluates reproducibility under a Slurm HPC environment.
Our practical objective is to identify low-compute improvements that remain compatible with a 2019-era training stack.
We first establish a reproducible baseline around BERT-based SBERT training and STSBenchmark evaluation,
then prepare an extensible runtime configuration for controlled comparison across model variants and data curricula.
This report summarizes the current pipeline and preliminary results.
\end{abstract}

\section{Introduction}
Sentence-BERT \cite{reimers-gurevych-2019-sentence-bert} addresses the inefficiency of cross-encoder sentence-pair
inference by learning fixed-size sentence embeddings that can be compared with simple similarity functions.
This design dramatically reduces pairwise search cost while retaining strong semantic textual similarity (STS) performance.

In this course project, we reproduce and extend the early SBERT setting using the original-style implementation
on sentence-transformers v0.2.4 (Dec 2019). The engineering target is not to maximize raw leaderboard performance with modern
large-scale pretraining, but to obtain measurable gains under constrained compute and reproducible Slurm execution.

Our current milestone focuses on:
\begin{itemize}
    \item Building robust Slurm scripts for data setup, training, and evaluation.
    \item Unifying runtime options via YAML configuration.
    \item Establishing a stable baseline before testing enhanced training objectives.
\end{itemize}

\section{Related Works}
The core reference is Sentence-BERT \cite{reimers-gurevych-2019-sentence-bert},
which adapts BERT \cite{devlin-etal-2019-bert} into a siamese/triplet framework for efficient
sentence embedding learning. The original paper reports strong transfer performance on STS tasks
with dramatically faster semantic search compared with vanilla BERT pairwise inference.

Our implementation scope currently follows the original SBERT spirit:
NLI/STS-driven supervised fine-tuning, pooling-based sentence embeddings, and STSBenchmark-based reporting.
The extension path in this project emphasizes lightweight training-strategy upgrades (e.g., multi-stage curricula)
that can be integrated into the legacy codebase with low engineering overhead.

\section{Experiment Setups}
\textbf{Codebase.}
We use a legacy Sentence-BERT repository aligned with sentence-transformers v0.2.4 (2019-style workflow).

\textbf{Execution environment.}
All experiments are executed using Single GPU NVIDIA L40S.

\textbf{Dataset protocol.}
The current evaluation target is STSBenchmark test split (\texttt{sts-test.csv}).
Training/evaluation scripts are configured to normalize expected dataset paths and optionally auto-download
required files when missing.

\textbf{Metrics.}
We report Pearson and Spearman correlations under cosine, Manhattan, Euclidean, and dot-product similarities.
For multi-seed evaluation, we aggregate mean and standard deviation over repeated runs.

\section{Methodology}
The current baseline pipeline follows the pretrained SBERT-style model evaluation protocol.
Given a model checkpoint (or model identifier), we encode sentence pairs from STSBenchmark and compute similarity scores.
The evaluator then reports correlation with gold STS labels.

Our runtime framework already supports configurable combinations of:
\begin{itemize}
    \item Model variant: SBERT-base / SBERT-large / SRoBERTa-base / SRoBERTa-large.
    \item Data mode: STS-only, NLI-only, STS+NLI, and STS+NLI+MNRL staged training.
    \item Epoch and seed controls through YAML and Slurm environment overrides.
\end{itemize}

At this stage, only one complete evaluation run group has finished. Additional training runs and ablations
will be filled into the same reporting format.

\section{Case Study: Failure Analysis}

Although MNRL improves STS in our \texttt{sts+nli+mnrl} setting, it does not consistently transfer to
all evaluation protocols. We observe two negative cases from completed experiments:
(1) a small but consistent degradation on Wiki Triplets retrieval accuracy, and
(2) a drop on SentEval transfer classification when comparing \texttt{nli+mnrl} against \texttt{nli}.

\begin{table}[ht]
\centering
\caption{Failure cases of MNRL on different tasks.}
\label{tab:failure-aggregate}
\begin{tabular}{lccc}
\hline
Benchmark / Metric & Baseline & +MNRL & $\Delta$ \\
\hline
Wiki Triplets & 80.80 & 80.42 & $-0.38$ \\
SentEval (mean) & 85.18 & 84.80 & $-0.38$ \\
\hline
\end{tabular}
\end{table}

For SentEval, the degradation is not limited to a single run: with seeds 42/43/44,
the run-level score differences are all negative ($-0.51$, $-0.31$, $-0.33$).
Task-wise differences are shown in Table~\ref{tab:failure-senteval-task}.

\begin{table}[ht]
\centering
\caption{SentEval task-level change (\texttt{nli+mnrl} minus \texttt{nli}, in points).}
\label{tab:failure-senteval-task}
\begin{tabular}{lc}
\hline
Task & $\Delta$ \\
\hline
MR Acc. & $-1.23$ \\
CR Acc. & $-0.40$ \\
SUBJ Acc. & $+0.86$ \\
MPQA Acc. & $-0.39$ \\
SST2 Acc. & $-1.26$ \\
TREC Acc. & $+0.27$ \\
MRPC Acc. & $-0.52$ \\
MRPC F1 & $-1.00$ \\
\hline
\end{tabular}
\end{table}

We attribute these failures to objective-task mismatch and data noise sensitivity.
MNRL emphasizes pairwise alignment with in-batch negatives, which is effective for similarity retrieval
but may weaken decision boundaries required by transfer classification tasks.
For Wiki Triplets, converting triplets to pair-only contrastive supervision can introduce noisy positives/negatives,
and the observed reduction across all three distance metrics suggests a broad embedding geometry shift rather than
an isolated metric artifact.
Finally, the Wiki comparison currently uses one run only, so the trend should be validated with multi-seed repetitions.
